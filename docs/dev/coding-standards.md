# コーディング規約（フロントエンド）

## 設計と依存
- 依存は最小限に保ち、標準 API と既存ユーティリティを優先します。新規ライブラリ追加時は目的・代替案・影響を明記してレビュー。
- ディレクトリとファイルは `kebab-case`。コンポーネントは責務単位で分割し、再利用を優先。
- 型安全を重視し、可能な限り明示的な戻り値と入力検証を行う（特に外部データ境界）。

## コードスタイル
- Lint/Format に従うことを前提に、可読性を重視した命名と早期 return を推奨。
- 副作用は関数の先頭でまとめて記述し、DOM/Canvas 操作は責務を分離する。
- ログは `console.error` を最小限に残し、デバッグログは開発時のみに限定。

## テストと検証
- UI・動作変更は `npm run build` / `npm run preview` を基本とし、再現手順と期待結果をドキュメント化。
- パフォーマンスが論点となる箇所は計測条件・指標（fps, bundle size 等）を記録する。
- デプロイ可否は `ops/deployment-checklist.md` の確認を必須とする。

## アクセシビリティと国際化
- テキストは `locales/` を経由し、直書きを避ける。ARIA 属性とラベルを明示する。
- カラーコントラストとフォーカス管理を確認し、キーボード操作を阻害しない。

## レビューとコミット
- 変更概要・意図・影響範囲をプルリクで説明し、関連ドキュメントを更新する。
- ドキュメント更新を含む場合は `DOC:` プレフィックスを付与し、本索引を更新する。
